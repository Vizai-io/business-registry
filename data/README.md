# Business Profile Data

This directory contains all business profiles in the VizAI Business Registry, organized by tier and category.

## Directory Structure

```
data/
├── index.json              # Master index (metadata, not a profile)
├── verified/               # Verified businesses
│   ├── technology/
│   │   ├── index.json     # Category index (metadata, not a profile)
│   │   └── vizai.json     # Business profile (validates against schema)
│   └── [other-categories]/
├── community/              # Community-submitted businesses
│   └── [categories]/
└── enterprise/             # Enterprise tier businesses
    └── [categories]/
```

## File Types

### Business Profiles (`*.json` except `index.json`)

These are actual business profiles that **validate against the business profile schema** (`schema/business-profile-v1.0.json`).

**Example:** `data/verified/technology/vizai.json`

**Structure:**
```json
{
  "schemaVersion": "1.0",
  "businessIdentifier": { ... },
  "description": { ... },
  "verification": { ... },
  "metadata": { ... }
}
```

### Index Files (`index.json`)

These are **metadata files** that provide fast discovery and querying. They do **NOT** validate against the business profile schema - they have their own structure.

**Locations:**
- `data/index.json` - Master index across all tiers
- `data/verified/[category]/index.json` - Index for each category

**Structure:**
```json
{
  "tier": "verified",
  "category": "technology",
  "lastUpdated": "2025-01-06",
  "count": 42,
  "profiles": [
    {
      "file": "technology/vizai.json",
      "domain": "vizai.io",
      "commonName": "VizAI",
      "lastVerified": "2025-01-06",
      "qualityScore": 100
    }
  ]
}
```

## Validation

Only business profiles (not index files) are validated:

```bash
# Validate a single profile
python tools/validation/validate-profile.py data/verified/technology/vizai.json

# Validate all profiles (excluding index files)
find data -name "*.json" -not -name "index.json" | \
  xargs -I {} python tools/validation/validate-profile.py {}
```

## Index Generation

Index files are automatically generated by the automation tool:

```bash
python tools/automation/generate-indexes.py
```

This scans all business profiles and creates index files for fast discovery.

## Categories

Available categories:
- **technology/** - Software, SaaS, hardware companies
- **professional-services/** - Consulting, legal, accounting
- **financial/** - Banks, fintech, investment firms
- **healthcare/** - Medical, pharmaceutical, health tech
- **manufacturing/** - Industrial, consumer goods
- **retail/** - E-commerce, brick-and-mortar retail
- **other/** - Businesses not fitting above categories

## Adding New Profiles

1. Create profile JSON in appropriate directory:
   ```bash
   data/verified/technology/newco.json
   ```

2. Validate against schema:
   ```bash
   python tools/validation/validate-profile.py data/verified/technology/newco.json
   ```

3. Regenerate indexes:
   ```bash
   python tools/automation/generate-indexes.py
   ```

4. Commit and push:
   ```bash
   git add data/verified/technology/newco.json data/**/index.json
   git commit -m "Add NewCo to verified registry"
   git push
   ```
